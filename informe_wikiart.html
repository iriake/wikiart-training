1. Introducción

1.1 Problema y Motivación

La clasificación y análisis de obras de arte es una tarea fundamental en la historia y crítica del arte, tradicionalmente realizada por expertos humanos basándose en características visuales, contexto histórico y análisis estético. Con el crecimiento exponencial de repositorios digitales de arte como WikiArt, surge la pregunta: ¿es posible utilizar técnicas de machine learning para identificar y caracterizar patrones visuales en obras de arte que correspondan con movimientos artísticos reconocidos?

Este proyecto aborda este problema desde una perspectiva de minería de datos, combinando análisis exploratorio exhaustivo con modelos predictivos y técnicas de clustering. Los hallazgos podrían tener implicaciones para:

  • Automatización de catalogación en museos y galerías
  • Descubrimiento de nuevas agrupaciones visuales no capturadas por la clasificación histórica
  • Identificación de características visuales que evolucionan temporalmente en el arte
  • Interpretabilidad de decisiones en sistemas de clasificación de arte

1.2 Alcance del Proyecto

Este informe documenta el Hito 1 de un proyecto de minería de datos que explora un dataset estructurado de 80,042 pinturas de WikiArt. El análisis comprende un análisis exploratorio completo, la ingeniería de características tabulares para todo el dataset y visuales básicas (brillo, contraste y colores en RGB/HSV) para una muestra de 3,000 imágenes, y el desarrollo de cuatro líneas de investigación independientes (clasificación, clustering, regresión e interpretabilidad) con modelos preliminares evaluados que permiten identificar los patrones y anomalías iniciales del conjunto de datos.

3. Preguntas y Problemas de Investigación

A partir del análisis exploratorio y del estado del arte, formulamos cuatro preguntas centrales acompañadas de sus hipótesis de trabajo:

Q1: Clasificación de Estilo Artístico

¿Qué tan bien se puede predecir el estilo artístico a partir de características geométricas y visuales básicas?

Hipótesis: Las propiedades geométricas de los lienzos (aspect ratio, escala logarítmica) y estadísticas de color manuales (promedio y desviación estándar RGB, saturación, brillo) contienen patrones correlacionados con corrientes artísticas. Sin embargo, la alta variabilidad estilística de los artistas limitará significativamente el rendimiento de clasificadores tradicionales basados en árboles cuando solo se usan datos tabulares.

Q2: Clustering Visual Alternativo

¿Existen agrupaciones naturales de pinturas basadas en descriptores visuales que difieran de la taxonomía histórica oficial?

Hipótesis: El clustering no supervisado revelará grupos homogéneos en cuanto a paleta de color y proporciones de lienzo, pero estos clusters mezclarán épocas y movimientos artísticos, evidenciando que ciertos patrones visuales trascienden las delimitaciones de escuelas históricas tradicionales.

Q3: Regresión Temporal

¿Es posible estimar de forma precisa el año de creación de una obra mediante características visuales y geométricas?

Hipótesis: Los formatos de los lienzos y las paletas han cambiado gradualmente con la introducción de nuevos pigmentos y tendencias artísticas. No obstante, las variables únicamente tabulares y geométricas no serán suficientes para modelar esta evolución temporal con precisión, arrojando una predicción cercana a la media histórica si no se integran variables visuales profundas.

Q4: Interpretabilidad en Atributos Clave

¿Cuáles son los atributos de composición espacial y tonalidad que mayor peso predictivo ejercen al distinguir los estilos?

Hipótesis: Atributos de formato de lienzo como aspect_ratio y la orientación espacial (portrait vs landscape) se alzarán como descriptores determinantes debido a los contrastes estructurales tradicionales (e.g. retratos clásicos y renacentistas verticales frente a paisajes impresionistas predominantemente horizontales).

4.2 Ingeniería de Características

Variables Tabulares (calculadas para el dataset de 80k pinturas)

  • aspect_ratio: Ancho dividido por el alto del lienzo.
  • log_width y log_height: Valores logarítmicos de las dimensiones para amortiguar outliers de alta resolución.
  • is_portrait: Indicador binario (1 si alto > ancho, 0 de lo contrario).
  • is_square: Indicador binario (1 si el aspect ratio está en el rango [0.9, 1.1]).
  • es_multilabel: Si el registro original poseía más de una etiqueta de género.

Variables Visuales Manuales (extraídas sobre una muestra representativa de 3,000 imágenes)

  • r_mean, g_mean, b_mean: Promedio tonal en los tres canales RGB (escala 0-1).
  • r_std, g_std, b_std: Desviación estándar por canal RGB (representa variabilidad cromática).
  • brightness: Brillo medio calculado mediante luminancia ponderada: \(0.299R + 0.587G + 0.114B\).
  • contrast: Desviación estándar del brillo.
  • saturation: Saturación promedio derivada del mapeo HSV.
  • warmth: Balance cálido-frío, cuantificado como la diferencia \(R_{mean} - B_{mean}\).

4.3 Procedimiento de Validación

Para asegurar la generalización de los resultados, se adoptaron las siguientes prácticas:

  • Partición estratificada de datos en proporción 80% entrenamiento y 20% prueba para clasificación.
  • Escalamiento estándar (StandardScaler) previo en algoritmos de clustering y regresión.
  • Métricas ponderadas (macro promedio) para clasificación para compensar el desbalance de clases de los estilos.

6. Conclusiones y Trabajo Futuro

6.1 Hallazgos Principales

  • Q1: Features tabulares solas tienen poder predictivo limitado (F1 ≈ 0.138), motivando la necesidad de deep learning o la incorporación de características visuales (F1 ≈ 0.236).
  • Q2: Los clusters visuales NO coinciden exactamente con géneros históricos, sugiriendo una taxonomía visual alternativa.
  • Q3: Las características tabulares no evolucionan de forma temporal lineal clara; el modelo de regresión arroja un MAE de 81.4 años lo cual representa solo un 0.2% de mejora sobre el baseline.
  • Q4: Las features más importantes son altamente interpretables y coherentes con la historia del arte.

6.2 Implicaciones

  • Para museos: Potencial para automatizar catalogación parcial, aunque se requiere validación humana.
  • Para historiadores del arte: Las agrupaciones visuales pueden revelar nuevas perspectivas sobre la evolución del arte.
  • Para machine learning: Los modelos interpretables (árboles de decisión) mantienen valor cuando se combinan con características significativas.

6.3 Limitaciones del Estudio

  • Solo 58.7% de las obras tienen año identificado.
  • Features visuales extraídas de muestra de 3k imágenes (3.75% del dataset).
  • No se incorporaron metadatos adicionales (contexto histórico, materiales, técnicas).
  • La variabilidad de resoluciones de imagen puede afectar las características visuales.

# Features tabulares derivadas de la geometría del lienzo

# Algoritmo de extracción de características visuales mediante Pillow y NumPy
from PIL import Image

def extract_visual_features(img_path):
    try:
        img = Image.open(img_path).convert('RGB').resize((128, 128))
        arr = np.array(img, dtype=np.float32) / 255.0

        # RGB stats
        r_mean, g_mean, b_mean = arr[:,:,0].mean(), arr[:,:,1].mean(), arr[:,:,2].mean()
        r_std, g_std, b_std = arr[:,:,0].std(), arr[:,:,1].std(), arr[:,:,2].std()

        # Brillo y contraste
        gray = 0.299*arr[:,:,0] + 0.587*arr[:,:,1] + 0.114*arr[:,:,2]
        brightness = gray.mean()
        contrast = gray.std()

        # Saturación (HSV) y Warmth
        cmax = arr.max(axis=2)
        cmin = arr.min(axis=2)
        saturation = np.where(cmax > 0, (cmax - cmin) / (cmax + 1e-8), 0).mean()
        warmth = r_mean - b_mean

        return {
            'r_mean': r_mean, 'g_mean': g_mean, 'b_mean': b_mean,
            'r_std': r_std, 'g_std': g_std, 'b_std': b_std,
            'brightness': brightness, 'contrast': contrast,
            'saturation': saturation, 'warmth': warmth
        }
    except Exception as e:
        return None

from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.tree import DecisionTreeClassifier
from sklearn.metrics import classification_report, f1_score

# Preparar conjuntos de datos sobre los top-10 géneros
top_genres = df['genre_principal'].value_counts().head(10).index.tolist()
df_clf = df[df['genre_principal'].isin(top_genres)].copy()

le_genre = LabelEncoder()
df_clf['label'] = le_genre.fit_transform(df_clf['genre_principal'])

X_tab = df_clf[['aspect_ratio', 'log_width', 'log_height', 'is_portrait', 'is_square', 'es_multilabel']]
y_clf = df_clf['label']

X_tr, X_te, y_tr, y_te = train_test_split(X_tab, y_clf, test_size=0.2, random_state=42, stratify=y_clf)

# Decision Tree Classifier
dt = DecisionTreeClassifier(max_depth=8, min_samples_leaf=20, random_state=42)
dt.fit(X_tr, y_tr)
print("F1 Decision Tree:", f1_score(y_te, dt.predict(X_te), average='macro'))

# Random Forest Classifier
rf = RandomForestClassifier(n_estimators=200, max_depth=15, random_state=42, n_jobs=-1)
rf.fit(X_tr, y_tr)
print("F1 Random Forest:", f1_score(y_te, rf.predict(X_te), average='macro'))
